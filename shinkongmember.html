工作項目

需求訪談｜Wireframe｜UI設計｜Guideline規範

專案角色

擔任產品設計| 設計2人＋工程師4人

專案介紹

本專案以「保戶專區 App」為核心，打造一站式保單管理平台，整合保單總覽、繳費、理賠、借款及文件申請等功能，協助保戶隨時掌握保單狀況並提升自主管理效率。整體 UI 與 User Flow 依據 Bebit 設計規劃書進行規劃，並透過與業主部門逐步協作，明確定義頁面內容與優化使用體驗。

設計過程

    依據 Bebit 規劃書的 Wireframe 與方向，完成實際 UI 設計並持續優化流程。透過壽險部門需求訪談，進一步簡化操作、完善細節，確保體驗更符合真實使用需求。

UI Guideline

Guideline定義整體畫面風格與細部元件，進行多人協作，並拆分功能同步設計，持續同步細部設計差異，確保整體方向不產生分歧。

RWD表格設計

解決RWD畫面中最令人頭痛的表格跑動方式，創造多種符合實際使用的表格形式，並製作元件，確保繁複的保險資訊在多裝置下閱讀順暢。

「簡單，才是真正的便利。」

良好的使用介面能帶來優質體驗，本專案的核心願景是打造「讓所有用戶都能快速上手的 App」。除了建立一致的設計規範外，更著重於 使用體驗提升，針對以下面向進行優化：
  • 易用性：固定主要操作按鈕位置，並依使用者視角重構資訊架構，讓功能更直覺易找。
  • 易讀性：凸顯主要功能，降低次要元素的視覺干擾，確保畫面焦點明確。
  • 舒適度：內頁與流程頁採 10% 彩度原則，以黑白灰為基調，降低視覺負擔。
  • 流程簡化：檢討並減化既有流程，搭配適當引導，讓使用者快速完成任務。
透過上述優化，產品不僅強化了一致性與清晰度，更有效提升了使用效率與整體體驗。
